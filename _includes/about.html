A different coverage

... that is actually useful.

Simple

Simply put, differential code coverage measures the changes in coverage between two revisions (e.g., between branches) and nothing else.

Focused

Coverage data unrelated to the changes at hand are discarded. With differential code coverage it's easy to see what's left to be tested.

Clutter-free

Traditional coverage reports display all lines of code and it is tedious to see if your changes were covered by tests. Not with couverture.io
